# 1. 要求仕様書 (Requirements Definition Document)

## 1.1 プロジェクト概要
* **プロジェクト名:** GIOSTRA メンバーシップ・コンシェルジュ・システム
* **目的:** LINE公式アカウントを通じた個別最適化メッセージの配信、およびAIによる顧客属性の自動抽出・管理。
* **ターゲット:** 経営者、ドクター、モデル、CA等のハイエンド層。

## 1.2 主要機能要求
1.  **AIプロファイリング機能:**
    * LINEチャット内の自然会話から「性別、年齢、職業、男女の好み、お酒の好み、居住地」をAIが自動抽出し、データベース（DB）へ記録する。
2.  **パーソナライズ配信機能:**
    * 配信時に、以下の要素を組み合わせた個別冒頭文をAIが自動生成する。
        * ニックネーム / 適切な季語 / 前回接触からの日数 / 過去の会話内容に基づいた一言。
3.  **セグメント抽出・フィルタリング:**
    * 「性別」「年齢制限」「特定の職業」によるターゲット抽出。
    * **NGリスト連携:** 特定のユーザーペアをNG設定し、配信対象から自動除外する。
4.  **マッチング履歴管理:**
    * イベント参加履歴に基づき、男女の「初対面か既面か」を自動判別し、カウントする。
5.  **モバイルファーストUI:**
    * PCを使わず、管理者のスマホ（LINE内リッチメニュー等）から全ての操作（抽出・生成・配信）を完結させる。

# 2. 設計仕様書 (Design Specification Document)

## 2.1 システム構成図（案）
* **Interface:** LINE Messaging API + LIFF (LINE Front-end Framework)
* **Backend:** Node.js または Python (FastAPI/Flask)
* **Database:** Firebase Realtime Database または PostgreSQL
* **AI Engine:** OpenAI API (GPT-4o or latest model)

## 2.2 データベース設計 (Data Schema)

### ユーザーテーブル (`Users`)
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `line_id` | String | LINE固有ID (PK) |
| `nickname` | String | ニックネーム |
| `gender` | Enum | 男 / 女 |
| `age` | Integer | 年齢（生年月日から算出） |
| `job_category` | String | 職業カテゴリ（経営者, 医療, モデル等） |
| `preferences` | JSONB | 男女の好み、お酒の好み |
| `location` | String | 居住地エリア |
| `last_interaction` | Date | 最終やり取り日 |
| `black_list` | Array | 遭遇NGなユーザーIDの配列 |

### イベント参加ログ (`Event_Logs`)
* `event_id`, `user_id`, `timestamp` を記録。
* **ロジック:** 配信リスト作成時、対象の男女ペアが過去に同一の `event_id` を持っているかをチェック。

## 2.3 AI 処理ロジック

### ① プロファイル抽出 (Extraction)
* **Trigger:** ユーザーからのメッセージ受信。
* **Prompt:** 「以下の会話から、[性別, 年齢, 職業, 好み, 居住地] をJSON形式で抽出せよ。既存データがある場合は更新せよ。」

### ② メッセージ生成 (Generation)
* **Input:** 共通イベント本文、ユーザープロファイル、前回接触日。
* **Prompt:** 「{nickname}様に対し、{季節/季語}を交え、{days}日ぶりの挨拶を作成せよ。過去に『{past_memo}』という話をしたことを踏まえ、自然な誘い文句にすること。トーンは上質で礼儀正しく。」

## 2.4 管理画面（LIFF）仕様
* **配信設定画面:**
    * スライダーによる年齢制限 (e.g. 25-40歳)。
    * 職業タグのチェックボックス選択。
    * 「NGリストを除外して配信」のトグルスイッチ。
* **プレビュー画面:**
    * 配信対象者数と、数名分のAI生成テキストを表示。OKなら一括送信。

# 3. 特記事項（開発上の留意点）
* **プライバシー:** AIによる分析を行う旨をプライバシーポリシーに明記すること。
* **精度の担保:** AIの誤抽出を防ぐため、管理者が手動でプロファイルを修正できるインターフェースを設けること。
* **コスト管理:** OpenAI APIのトークン消費量を抑えるため、プロファイル抽出は「情報の更新があった時のみ」実行する設計にすること。
